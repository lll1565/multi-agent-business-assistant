# API 文档查询技能

## 工作流程

1. **明确接口** — 用户说的是哪个 API（名称或路径关键词）？
2. **列出或查询**
   - 不确定有哪些接口 → `list_all_apis`
   - 已知名称或关键词 → `query_api_doc`
3. **格式化输出** — 使用【API文档】标题，包含方法、路径、参数、响应示例

## 规则

- 只使用工具返回的文档，不要编造端点
- 多个匹配时全部列出并说明区别
- 用户用中文提问时用中文回答
